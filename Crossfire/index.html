Overview

CrossFire is a universal package manager CLI tool designed to unify package management across different operating systems and package ecosystems. Version 3.0.0 represents a stable, production-ready implementation that intelligently detects available package managers and provides a consistent interface for package operations.

Key Features

  • Universal Package Management: Install packages across 13+ different package managers
  • Intelligent Manager Selection: Automatically chooses the best package manager for each package
  • Cross-Platform Support: Works on Windows, macOS, and Linux distributions
  • Concurrent Operations: Multi-threaded downloads and parallel manager updates
  • Self-Updating: Built-in update mechanism with integrity verification
  • System Health Monitoring: Comprehensive health checks and statistics
  • Robust Error Handling: Retry mechanisms and graceful failure recovery
  • JSON Output Mode: Machine-parseable output for automation

Architecture & Design

Core Architecture

CrossFire follows a modular architecture with clear separation of concerns:

CrossFire
├── Core Components

│   └── Configuration Management

└── Advanced Features
    ├── Self-Update System
    ├── Health Monitoring
    ├── Statistics Collection
    └── System Cleanup

Design Principles

  1. Fail-Safe Operations: Always attempt graceful degradation
  2. User Experience First: Clear, actionable feedback for all operations
  3. Platform Agnostic: Write once, run everywhere approach
  4. Security by Default: Validate all external inputs and downloads
  5. Performance Optimized: Use concurrency where beneficial
  6. Extensible: Easy to add new package managers and features

Threading Model

CrossFire uses Python's concurrent.futures.ThreadPoolExecutor for:

  • Parallel package manager status checks
  • Concurrent manager updates (max 4 workers)
  • Multi-chunk downloads for large files
  • Bulk package operations

Installation & Setup

Prerequisites

  • Python 3.7+ (CrossFire is written in Python)
  • Internet connection (for downloads and updates)
  • Administrator/root privileges (for some package managers)

Quick Start

  1. Download CrossFire: 
    curl -O https://raw.githubusercontent.com/BCAS-Team/CrossFire/main/CrossFireL/crossfire.py
  2. Make executable (Unix-like systems): 
    chmod +x crossfire.py
  3. Run initial setup: 
    python crossfire.py --setup

What --setup Does

The setup process performs several critical operations:

  1. PATH Configuration:
    • Detects your shell (bash, zsh, fish)
    • Adds CrossFire directory to PATH in appropriate RC file
    • Handles shell-specific syntax differences
  2. Launcher Installation:
    • Unix/Linux/macOS: Creates executable script in ~/.local/bin or /usr/local/bin
    • Windows: Creates .bat file in script directory
    • Tests launcher functionality before confirming success
  3. Shell Detection Logic: 
    def _get_shell_rc_file() -> str:
    shell = os.environ.get("SHELL", "")
    home = os.path.expanduser("~")

    if shell.endswith("zsh"):
        return os.path.join(home, ".zshrc")
    elif shell.endswith("bash"):
        return os.path.join(home, ".bashrc")  # Linux
        # or .bash_profile on macOS
    elif shell.endswith("fish"):
        return os.path.join(home, ".config", "fish", "config.fish")
    else:
        return os.path.join(home, ".profile")  # Fallback

Manual Installation

For environments where automatic setup isn't suitable:

  1. Place script in desired location
  2. Add to PATH manually: 
    export PATH="/path/to/crossfire:$PATH"
  3. Create alias (alternative): 
    alias crossfire='python /path/to/crossfire.py'

Command Execution System

RunResult Dataclass

@dataclass
class RunResult:
    ok: bool  # True if command succeeded (return code 0)
    code: int # Actual return code
    out: str  # stdout content
    err: str  # stderr content

Command Parsing

The _split_cmd function handles both string and list command formats:

def _split_cmd(cmd: str | List[str]) -> List[str]:
    if isinstance(cmd, list):
        return cmd

    import shlex
    try:
        return shlex.split(cmd)  # Properly handle quoted arguments
    except ValueError:
        return cmd.split()  # Fallback for simple cases

Execution Features

  • Timeout Protection: Prevents hanging on unresponsive commands
  • Retry Mechanism: Exponential backoff for transient failures
  • Environment Control: Isolated environment variables
  • Shell Detection: Automatic shell usage for complex commands
  • Cross-Platform: Handles Windows-specific creation flags

Package Manager Integration

Integration Model

CrossFire uses a factory pattern to handle different package managers. Each manager (e.g., APT, NPM, Pip) is a separate class with a standardized interface for installation, removal, and other operations.

Example: APT Integration

class AptManager(PackageManager):
    def __init__(self):
        super().__init__("apt", "Debian/Ubuntu", "system", ["apt", "apt-get"])
        
    def install(self, package_name: str, options: dict) -> RunResult:
        cmd = f"sudo apt-get install -y {package_name}"
        return run_command(cmd)

    # ... other methods like remove, update, search

Manager Selection Logic

The core logic for manager selection is a priority-based system:

  1. Explicit Manager: If --manager flag is used, that manager is prioritized.
  2. File-Based Detection: If a manifest file is found (e.g., requirements.txt, package.json), the corresponding manager is chosen.
  3. Heuristic Scoring: A scoring system based on platform, package name, and user preferences determines the best available manager.
  4. Default Fallback: If no manager is selected, a pre-defined default is used (e.g., Homebrew on macOS, APT on Debian).

System Detection

Accurate system detection is crucial for CrossFire's cross-platform functionality. It determines available package managers, command syntax, and installation paths.

Detection Flow

  1. OS Detection: Uses platform.system() to identify the base OS (Linux, Windows, Darwin).
  2. Distribution Detection: On Linux, uses the distro library to get detailed distribution info (e.g., Ubuntu, Fedora).
  3. Architecture: Checks for 32-bit or 64-bit architecture to handle system-specific binaries.
  4. Shell Detection: Determines the user's current shell (e.g., bash, zsh, PowerShell) for correct path and alias configuration.

Advanced Features

Self-Update System

CrossFire includes a robust self-update mechanism to ensure users are always running the latest version with security patches and new features.

  • Integrity Check: Downloads are verified with a SHA256 checksum.
  • Atomic Update: The update process is designed to be atomic, preventing corruption if interrupted.
  • Rollback: A backup of the previous version is kept for easy rollback.

System Health Monitoring

The --health command provides a quick overview of the system state, including disk space, memory usage, and the status of core CrossFire components.

Statistics Collection

With user consent, CrossFire can send anonymized statistics about package installations to improve the intelligent manager selection algorithm.

Configuration

Users can customize CrossFire's behavior via a crossfire.conf file located in the user's home directory. This file uses a simple INI format.

Example crossfire.conf

[core]
default_manager = "apt"
json_output = false

[updates]
auto_update = true
check_interval_days = 7

[telemetry]
enabled = false

The configuration can also be overridden with command-line flags.

Error Handling & Recovery

CrossFire is built to be resilient. It employs a multi-layered approach to error handling.

  • Command Retries: Transient errors (e.g., network timeout) trigger an exponential backoff retry.
  • Graceful Degradation: If a specific package manager fails, CrossFire attempts to use an alternative manager for the same package.
  • Actionable Feedback: Error messages are designed to be human-readable and provide clear, actionable steps for the user.

Example Error Log

[ERROR] Operation failed for package 'numpy' using manager 'pip'.
[REASON] Network connection timed out.
[ACTION] Retrying in 5 seconds... (Attempt 2/3)

Security Considerations

Security is a primary concern. CrossFire follows several best practices to mitigate risks.

  • Input Validation: All command-line inputs and configuration values are sanitized.
  • Binary Integrity: The self-update mechanism uses digital signatures and checksums.
  • Privilege Management: The tool runs with the least privilege required for each operation, escalating with sudo or UAC only when necessary.
  • Dependency Scanning: Integrates with external tools to scan for known vulnerabilities in installed packages.

Performance & Optimization

CrossFire is optimized for speed and efficiency to provide a smooth user experience.

  • Caching: Caches results of system checks and package manager status.
  • Concurrency: Uses parallel threading for non-blocking operations.
  • Memory Management: Efficiently handles large data streams from package manager outputs.

Troubleshooting

If you encounter issues, here are some common solutions:

  1. Check for prerequisites: Ensure Python 3.7+ is installed.
  2. Run with verbose flag: Use crossfire -v to get detailed logs.
  3. Check configuration: Verify the contents of your crossfire.conf file.
  4. Re-run setup: The --setup flag can fix broken PATH configurations.

Development & Extension

The modular design makes it easy to contribute to CrossFire.

Adding a New Package Manager

To add a new manager, you typically need to:

  1. Define its properties in the central managers dictionary (e.g., name, platform support, commands).
  2. Implement a function to handle its installation, removal, or update logic.
  3. (Optional) Add a heuristic function to help CrossFire intelligently select it for certain package types.
